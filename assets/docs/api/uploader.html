headers 信息

是否自动上传 设置为 true 后，不需要手动调用 upload，有文件选择即开始上传。

Properties

NameTypeDefaultDescription

服务端网址

HTTP请求方式

aliasstringfile

设置文件上传域的name

withCredentialsbooleantrue

是否发送凭据

headersUploaderHeaders[]

headers 信息

params{ [key: string]: any }

HTTP提交其他参数

filtersFilterFunction[]

过滤器

autobooleanfalse

是否自动上传 设置为 true 后，不需要手动调用 upload，有文件选择即开始上传。

mimesstring[]

限定文件mime类型，例如：[ '' ]

typesstring[]

限定文件类型，例如：[ 'jpg', 'png' ]

limitnumber-1

允许最多上传数量 -1 表示不受限

sizenumber-1

限定文件大小（单位：字节） -1 表示不受限

removeAfterUploadbooleanfalse

是否自动移除上传成功文件

disableMultipartbooleanfalse

禁止使用multipart/form-data

onFileQueuedFunction

当文件被加入队列以后触发

onFileDequeuedFunction

当文件被移除队列后触发

onStartFunction

当开始上传流程时触发。

onCancelFunction

当开始上传流程取消时触发。

onFinishedFunction

当所有文件上传结束时触发

onUploadStartFunction

某个文件开始上传前触发，一个文件只会触发一次。

onUploadProgressFunction

上传过程中触发，携带总的上传进度，以及当前文件的上传进度

onUploadSuccessFunction

当文件上传成功时触发

onUploadErrorFunction

当文件上传出错时触发

onUploadCompleteFunction

不管成功或者失败，文件上传完成时触发

onUploadCancelFunction

取消某文件时触发

onErrorFunction

当filters不通过时触发

内置的上传组件是基于HTML5 如有特殊需求可以自定义上传接口（Observable中的 any 指的是事件当中的response） 不管成功与否都会触发 onUploadComplete & onUploadSuccess
